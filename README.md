# dotfiles

nikroulah's personal Mac setup, organized by topic. This was inspired by [holman](https://github.com/holman/dotfiles)'s, but is maybe overkill since I'm not that opinionated about stuff. I just wanted to try and, as much as I can, avoid the long tail of discovering that various things aren't set up the way I'm used to. Really this is less of a dotfiles (there is like one or two dotfiles at the time of writing) repo and more of a new laptop bootstrap thing that installs various things that I am used to having, though maybe that will change over time.

## Install

```sh
git clone git@github.com:nikroulah/dotfiles.git ~/dotfiles
cd ~/dotfiles
script/bootstrap   # first-time: Xcode CLT, git identity, symlinks
bin/dot            # macos defaults + homebrew + every topic's install.sh, then prints reminders
```

This will spit out a list of reminders and potentially manual steps that need to happen.

Day-to-day, `dot` (in `bin/`, on PATH after install) refreshes everything: regenerates symlinks, applies `macos/set-defaults.sh`, updates homebrew, upgrades packages, re-runs the topic installers, and re-prints reminders. `dot -e` opens this repo in `$EDITOR`.

## Conventions

Anything in a topic directory matching these names gets picked up automatically:

- **`<topic>/install.sh`** — run by `script/install` (alphabetical, homebrew first). Should be idempotent and safe/fast to re-run.
- **`<topic>/*.symlink`** — symlinked into `$HOME` as `.<name>` by `script/bootstrap` (e.g. `git/gitconfig.symlink` → `~/.gitconfig`). Prompts before touching existing files.
- **`<topic>/reminders.md`** — manual steps that can't be scripted (permissions pop-ups, browser auth, etc.). Printed by `script/reminders` at the end of `script/install`.
- **`bin/`** — added to `$PATH` by `fish/install.sh`; anything here is runnable anywhere (currently just `dot`).

## Machine-local config

Git identity lives in `~/.gitconfig.local`, written by `script/bootstrap` (it prompts) and included from `gitconfig.symlink` — so work machines can get the work-specific email without touching the repo.
