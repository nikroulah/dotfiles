# dotfiles

nikroulah's personal Mac setup. This was inspired by [holman](https://github.com/holman/dotfiles)'s, but is maybe overkill since I'm not that opinionated about stuff. I just wanted to try, as much as I can, toavoid the long tail of discovering that various things aren't set up the way I'm used to. Really this is less of a dotfiles repo (there are like two or three dotfiles at the time of writing) and more of a new laptop bootstrap thing that installs various things that I am used to having, though maybe that will change over time.

## Install

```sh
git clone git@github.com:nikroulah/dotfiles.git ~/dotfiles
cd ~/dotfiles
script/bootstrap   # first-time: Xcode CLT, git identity, symlinks
bin/dot            # macos defaults + homebrew + every topic's install.sh, then prints reminders
fish               # 🐟
```

This will spit out a list of reminders and potentially manual steps that need to happen.

You can run `dot` (which is on `PATH` after install) periodically to rerun everything: generate symlinks, set macos defaults `macos/set-defaults.sh`, update brew packages, re-run installers, re-print reminders. You can also skip steps with e.g. `dot --skip-install --skip-brew`, or `dot -e` to open this `dotfiles` dir in `$EDITOR` (`vim` by default). You can see `dot --help` for more info.

## Conventions

Anything in a topic directory matching these names gets picked up automatically:

- **`<topic>/install.sh`** — for installing things (alphabetical by topic, homebrew first). Should be idempotent.
- **`<topic>/*.symlink`** — will have a symlink to it created in `$HOME` as `.<name>` (e.g. `git/gitconfig.symlink` → `~/.gitconfig`). Prompts before touching existing files.
- **`<topic>/reminders.md`** — manual steps that can't be scripted (permissions pop-ups, browser auth, etc.), which get printed out after the install step.
- **`bin/`** — gets added to `$PATH` in `fish/install.sh`; anything here is runnable anywhere (currently just `dot`).

## Machine-local config

Git identity lives in `~/.gitconfig.local`, written by `script/bootstrap` (it prompts) and included from `gitconfig.symlink` — so e.g. work machines can get the work-specific email without touching the repo.
